# 乒乓球动作对比分析系统 — 产品需求文档（PRD）

## 1. 项目信息

| 字段 | 内容 |
|------|------|
| Language | 中文 |
| Programming Language | 单页面 HTML（内嵌 CSS + JS）+ MediaPipe Pose（CDN） |
| Project Name | `pingpong-motion-compare` |
| 原始需求 | 在已有乒乓球 AI 教学系统基础上，新增摄像头实时捕捉学员动作、与马龙标准动作对比、AI 生成调整建议、针对性训练计划四大功能 |

---

## 2. 产品定义

### 产品目标

**一句话描述**：通过摄像头实时捕捉学员乒乓球动作，与马龙标准动作进行骨骼关键点对比分析，自动生成调整建议和个性化训练计划。

**3 个核心指标**：

1. **关键点识别准确率** — MediaPipe Pose 对正面/侧面视角的 33 个骨骼关键点提取成功率 ≥ 90%
2. **动作对比反馈延迟** — 从动作采集到差异分析结果展示 ≤ 3 秒（单次动作片段）
3. **训练建议覆盖率** — 对比分析后至少输出 5 项可执行的调整建议，覆盖主要技术环节

### 用户故事

1. **As a** 乒乓球学员，**I want** 通过摄像头实时查看自己的动作画面并叠加骨骼线，**so that** 我能直观了解自己的身体姿态和动作轨迹。

2. **As a** 乒乓球学员，**I want** 将我的动作与马龙的标准动作进行逐帧对比，**so that** 我能精确看到哪些关节角度存在偏差。

3. **As a** 乒乓球学员，**I want** 系统自动生成调整问题清单（如"引拍肘角偏大 15°"），**so that** 我能有针对性地纠正错误动作。

4. **As a** 乒乓球学员，**I want** 根据调整建议获得个性化训练计划（如专项练习、推荐时长），**so that** 我能高效安排后续训练。

5. **As a** 乒乓球学员，**I want** 在对比分析页面同时查看自己的实时画面和标准动作回放，**so that** 我可以在练习中即时对照调整。

## 3. 技术规范

### 需求池

#### P0 — Must Have（必须实现）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| P0-1 | 摄像头实时接入 | 通过 `getUserMedia` 获取摄像头视频流，在页面中实时显示画面；支持前置/后置切换（移动端）；权限被拒绝时给出友好提示 |
| P0-2 | MediaPipe Pose 骨骼识别 | 加载 MediaPipe Pose（CDN），实时提取 33 个骨骼关键点；在视频画面上绘制骨骼连线和关键点；帧率 ≥ 15 FPS |
| P0-3 | 马龙标准动作数据 | 以 JSON 格式内嵌马龙正手攻球、反手拨球、发球 3 个标准动作的关键点序列（每个动作 10-20 帧关键帧），数据按 MediaPipe 33 点坐标格式组织 |
| P0-4 | 动作对比分析 | 用户录制一段动作（3-10 秒），系统提取关键帧，与对应标准动作的关键帧进行角度差异计算；输出各关节角度差异值和综合相似度评分 |
| P0-5 | 调整建议生成 | 基于关键点角度差异，按规则引擎生成调整问题清单（如"肘关节角度偏大 15°，建议减小引拍幅度"）；至少覆盖肩、肘、腕、髋、膝 5 个关节维度 |
| P0-6 | 新增侧边栏导航项 | 在现有系统侧边栏"主菜单"区域新增"动作对比"导航项，图标 `fa-solid fa-people-arrows`，点击进入动作对比页面 |

#### P1 — Should Have（应该实现）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| P1-1 | 个性化训练计划 | 根据调整建议中各问题的优先级和类型，生成 1-4 周的针对性训练计划，包含训练项目、频次、时长建议 |
| P1-2 | 动作类型选择 | 用户可选择对比的动作类型（正手攻球 / 反手拨球 / 发球），系统加载对应的马龙标准动作数据 |
| P1-3 | 对比结果可视化 | 在对比结果区域以左右分屏或叠加方式展示学员动作骨骼与标准动作骨骼；差异关节用红色高亮标注 |
| P1-4 | 分析历史记录 | 将每次对比分析结果（相似度评分、调整建议）保存至 localStorage，可在页面内查看历史对比趋势 |

#### P2 — Nice to Have（可以实现）

| 编号 | 需求 | 验收标准 |
|------|------|----------|
| P2-1 | 实时骨骼对比模式 | 录制时不只显示自己骨骼，同时在画面旁叠加半透明的标准动作骨骼，实现边练边对照 |
| P2-2 | 动作评分趋势图 | 在成长复盘页面增加"动作对比评分趋势"图表，展示历次对比分析的综合评分变化 |
| P2-3 | 导出对比报告 | 支持将单次对比分析结果导出为图片（Canvas 截图），方便分享给教练 |
| P2-4 | 更多标准动作 | 增加正手拉弧圈、搓球、挑打等更多标准动作的关键点数据 |

### 4.2 交互流程

1. **进入页面** → 显示摄像头预览（左屏）+ 马龙标准动作静止帧（右屏）
2. **选择动作类型** → 右屏切换为对应标准动作的骨骼动画
3. **点击「开始录制」** → 倒计时 3 秒后开始录制，左屏显示实时骨骼叠加
4. **录制完成（3-10 秒）** → 自动进行对比分析
5. **展示分析结果** → 显示综合相似度、各关节差异、AI 调整建议
6. **生成训练计划** → 基于建议自动生成，用户可调整周期（1-4 周）

### 4.3 移动端适配

- 视频区域由左右分屏改为上下分屏（我的动作在上，标准动作在下）
- 动作类型选择改为下拉选择器
- 调整建议和训练计划可折叠展开

### 4.4 与现有系统的集成点

| 集成点 | 说明 |
|--------|------|
| 侧边栏导航 | 新增"动作对比"项，位于"动作分析"与"成长复盘"之间 |
| 移动端底部导航 | 新增"对比"tab，5 项改为 6 项 |
| 训练记录 | 对比分析完成后可一键保存为训练记录，自动填充动作类型和 AI 评分 |
| 成长复盘 | P2 阶段在成长复盘页面增加"动作对比评分趋势"图表 |
| 仪表盘 | AI 每日建议卡片中可引用最近一次动作对比的关键发现 |

## 5. 待确认问题

| 编号 | 问题 | 影响范围 | 建议默认值 |
|------|------|----------|-----------|
| Q1 | 马龙标准动作数据来源 — 是使用公开视频人工标注关键点，还是用 AI 从视频中提取？ | P0-3 数据质量 | 先用 AI 从视频中提取 + 人工微调，内嵌为静态 JSON |
| Q2 | 录制时长是否固定？允许用户自定义录制时长还是固定 3 秒？ | P0-4 交互设计 | 默认 3 秒，提供 3s / 5s / 10s 三档选择 |
| Q3 | 动作对比时，学员应从正面还是侧面面对摄像头？不同角度的关键点差异算法不同 | P0-4 算法设计 | 优先支持侧面视角（最能体现引拍-击球-随挥的动作轨迹），正面作为辅助 |
| Q4 | 是否需要支持视频上传（非实时）方式的动作对比？ | P1 范围 | P1 阶段暂不支持，仅支持实时摄像头录制 |
| Q5 | 训练计划的保存 — 是否需要保存到 localStorage 与现有训练记录打通？ | P1-1 数据设计 | 是，训练计划保存后可在"课程计划"页面查看 |
| Q6 | 马龙标准动作的 3 个动作（正手攻球、反手拨球、发球）是否满足 MVP 范围？ | P0-3 数据范围 | 是，MVP 先支持这 3 个动作，后续可扩展 |
